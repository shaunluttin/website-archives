Setup MiniBlog with GitHub and Windows Azure

Tuesday, April 29, 2014 4:31:38 AM

MiniBlog is awesome. Here’s one way to get started.

Prerequisites

  • A Windows Azure account.
  • A GitHub account.
  • GitHub for Windows.

    Once you have those, follow these steps.

    Fork MiniBlog on GitHub

  • Log into GitHub.
  • Then go to https://github.com/madskristensen/miniblog
  • Then click Fork (top right corner) to obtain your own repo of MiniBlog

    Setup a Windows Azure Website

  • Log into Windows Azure. Then…
  • Click WEB SITES > NEW > CUSTOM CREATE
  • The Create Web Site window will open.
  • Add a URL.
  • Check Publish from source control.
  • Click Next.

  • The Where is your source code? window will open.
  • Choose GitHub
  • The Choose a repository to deploy window will open.
  • Choose the MiniBlog repo that we just forked.

  • Confirm and wait one minute for Azure to publish and deploy.

    You can now visit the blog at your chosen URL (e.g. mini-blog.azurewebsites.net). It will have the default mini-blog content, username, password, and theme.

    Change Your Password

  • Log back into your GitHub account.
  • Clone your fork of MiniBlog from GitHub to your computer.

  • Once it completes cloning, open GitHub for Windows.
  • Right click on the MiniBlog repository and choose Open in explorer.
  • Open the Web.config file.
  • Edit the authentication section.
  • Replace the user name with your own.
  • Replace the password with a hash generated here: http://www.stringfunction.com/sha1-hash.html

    My Finished Authentication Section

    <authentication mode="Forms">
  <forms defaultUrl="~/" loginUrl="~/views/login.cshtml" name="miniblog" timeout="10080">
    <credentials passwordFormat="SHA1">
      <!-- Password is "demo". Generate your hash password here http://www.stringfunction.com/sha1-hash.html -->
      <user name="demo" password="0898a7e586406b7f9ec379be32d45f9b33348c3e"/>
    </credentials>
  </forms>
</authentication>

    Further Customizations Are Up to You…

    • Delete Blog Posts. Do this by deleting files that are in the “posts” directory.
    • Setup Live Writer. This works beautifully right out of the box.
    • Customize a Theme.
    • Setup the Comments Emailer. 
    • Approve a Comment.
    • Et Cetera…
